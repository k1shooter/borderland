# BORDERLAND WebApp

아리스 인 보더랜드 감성의 **실시간 데스게임 웹앱 프로토타입**입니다.

이 프로젝트는 다음을 포함합니다.
- 4무늬 × 1~10 난이도 = **총 40개 트럼프 카드 게임 카탈로그**
- **실시간 로비 / 방 생성 / 준비 / 시작 / 채팅**
- **사망(탈락) 처리 후 재접속 차단**
- **어드민 계정**
- **리더보드**
- **스페이드 전용 웹 피지컬 미니게임 10종**
- 업로드된 `game.txt`를 바탕으로 확장한 룰북
- 선택형 **블록체인 사망 레지스트리 컨트랙트**

## 기술 스택
- Backend: Node.js, Express, Socket.IO
- Frontend: Vanilla JS, HTML, CSS, Canvas, Web Audio API
- Storage: JSON 파일 기반 프로토타입 저장소
- Optional: Solidity 스마트 컨트랙트

## 실행 방법
```bash
npm install
npm start
```

브라우저에서 아래 주소를 여세요.
```bash
http://localhost:3100
```

## 기본 어드민 계정
- ID: `admin`
- PW: `borderland-admin-2026!`

## 주요 폴더
- `public/` : 프론트엔드
- `lib/` : 게임 엔진, 스토리지, 유틸
- `shared/gameCatalog.json` : 40개 카드 게임 메타데이터
- `docs/GAME_RULEBOOK.md` : 전체 게임 규칙집
- `docs/SYSTEM_GUIDE.md` : 시스템 구조 설명
- `docs/SECURITY_NOTES.md` : 사망 잠금 / IP / 지갑 / 블록체인 관련 노트
- `contracts/DeathRegistry.sol` : 선택형 온체인 사망 레지스트리

## 포함된 기능
### 1) 계정 / 사망 잠금
- 회원가입 / 로그인
- 탈락 시 `DEAD` 처리
- 재로그인 차단 기준
  - 계정
  - IP 해시
  - 디바이스 ID
  - 지갑 주소 해시(입력 시)
- 주의: 현재 IP 기반 잠금은 요청사항을 반영한 것으로, 실제 서비스에서는 공유망/PC방/학교망 오탐이 생길 수 있습니다.

### 2) 실시간 룸 시스템
- 공개 방 생성
- 카드 선택
- 참가자 Ready
- 호스트 시작
- 봇 채우기(테스트용)
- 실시간 채팅
- 카드별로 채팅 가능/제한 정책 분리

### 3) 게임 엔진
- 스페이드: 피지컬 미니게임 10종
- 클로버: 협동/합의/분담형 게임 10종
- 다이아: 퍼즐/추론/연산형 게임 10종
- 하트: 배신/심리/투표형 게임 10종

### 4) 리더보드
- 생존하고 통과한 카드 수를 기준으로 순위 집계
- 획득 카드 목록 저장

### 5) 어드민 기능
- 전체 게임 테스트 가능
- 죽은 유저 리셋 API 제공

## 현재 구현 수준
이 프로젝트는 **실행 가능한 MVP / 프로토타입**입니다.

즉, 아래는 이미 들어가 있습니다.
- 서버 실행
- 기본 인증
- 실시간 소켓 이벤트
- 룸 / 채팅 / 게임 세션 흐름
- 40개 카드 카탈로그
- 각 카드에 대응하는 엔진 핸들러
- 피지컬 게임용 브라우저 미니게임 렌더러

다만, 실제 대규모 서비스 수준으로 가려면 아래 보강이 필요합니다.
- DB(PostgreSQL/Redis)로 전환
- 리플레이 방지 / 안티치트 강화
- WebRTC 음성/영상 연동(선택)
- 서버 권한형 판정 확대
- 스케일아웃 구조
- 실지갑 로그인/서명 검증
- 정교한 블록체인 연동
